# Data Formats — All JSON Files

Every data file used by the pipeline is documented here: purpose, producer, consumers,
schema, and lifecycle.

---

## File inventory

| File | Written by | Read by | Volatility |
|---|---|---|---|
| `iv_history.json` | iv_backfill, iv_tracker | iv_tracker | Updated daily |
| `iv_rank_cache.json` | iv_backfill, iv_tracker | screener, analyzer | Updated daily |
| `options_candidates.json` | options_screener | selector, analyzer | Replaced daily |
| `options_picks_history.json` | options_screener | humans, dashboard | Appended daily |
| `options_pending_entries.json` | options_strategy_selector | options_executor | Replaced each run |
| `positions_state.json` | options_executor, options_monitor | monitor, analyzer, optimizer | Updated on entry/exit |
| `options_signal_quality.json` | options_signal_analyzer | options_optimizer | Replaced daily |
| `options_improvement_report.json` | options_optimizer | humans, dashboard | Replaced daily |
| `alpaca_config.json` | Human (manual) | All modules with API calls | Never auto-changed |
| `options_config.json` | Human + optimizer | All modules | Updated by optimizer (n≥50) |

## `iv_history.json`

**Purpose:** 252-day rolling history of IV (or HV30 proxy) for every symbol.
The source of truth for IV rank computation.

**Structure:**
```json
{
  "AAPL": {
    "2025-09-10": 0.312,
    "2025-09-11": 0.298,
    "...": "...",
    "2026-04-25": 0.341
  },
  "TSCO": {
    "2026-04-25": 0.445
  }
}
```

Keys: symbol (string) → date (ISO-8601 string) → IV as decimal fraction (0.30 = 30%).

**Lifecycle:**
- Created by `iv_backfill.py` on first run with ~157 proxy dates per symbol.
- Appended daily by `iv_tracker.py` with one real snapshot date per symbol.
- Proxy dates phase out of the 252-day rolling window after ~12 months as real data accumulates.
- File grows by ~512 entries per day (one per symbol).

**No strict schema validation.** Missing dates are silently skipped. Symbols not in the file
get `iv_rank = None` in the cache.

## `iv_rank_cache.json`

**Purpose:** Fast-lookup cache of today's IV rank (and related metrics) per symbol.
Computed from `iv_history.json`. The screener reads this rather than recomputing IV rank
from history on every run.

**Structure:**
```json
{
  "AAPL": {
    "iv_rank":        72.4,
    "iv_percentile":  68.1,
    "current_iv":     0.341,
    "min_iv_252d":    0.198,
    "max_iv_252d":    0.512,
    "updated_at":     "2026-04-25"
  },
  "TSCO": {
    "iv_rank":        100.0,
    "current_iv":     0.445,
    "updated_at":     "2026-04-25"
  }
}
```

**IV rank formula:** `(current_iv - min_252d) / (max_252d - min_252d) * 100`

Range: 0–100. Higher = options more expensive than historical average.

**Lifecycle:** Fully rewritten by `iv_tracker.py` (and `iv_backfill.py`) each run.
Stale if `iv_tracker` was skipped with `--no-iv`.

## `options_candidates.json`

**Purpose:** Today's screened candidates — stocks that passed all entry filters.
Passed to the strategy selector for contract selection.

**Structure:**
```json
{
  "generated_at": "2026-04-25T21:14:12+00:00",
  "regime": "bull",
  "candidates": [
    {
      "symbol":        "TSCO",
      "price":         214.50,
      "rsi":           20.5,
      "volume_ratio":  1.99,
      "iv_rank":       100.0,
      "current_iv":    0.445,
      "strategy":      "CSP",
      "near_earnings": false,
      "signal_score":  61.4
    }
  ]
}
```

**Lifecycle:** Fully replaced on every screener run. Safe to delete (regenerated next run).

## `options_picks_history.json`

**Purpose:** Permanent research log — every candidate ever generated by the screener.
Not used by any automated module; intended for human review and dashboard display.

**Structure:**
```json
[
  {
    "date":          "2026-04-25",
    "symbol":        "TSCO",
    "price":         214.50,
    "rsi":           20.5,
    "iv_rank":       100.0,
    "strategy":      "CSP",
    "signal_score":  61.4,
    "regime":        "bull"
  }
]
```

**Lifecycle:** Appended each screener run. Never truncated by the pipeline.
Grows indefinitely.

## `options_pending_entries.json`

**Purpose:** Contracts selected by the strategy selector, queued for the executor.
One entry per candidate that passed contract validation (OI, spread, delta targeting).

**Structure:**
```json
[
  {
    "symbol":          "TSCO",
    "strategy":        "CSP",
    "expiry":          "2026-06-19",
    "dte":             55,
    "strike":          195.0,
    "delta":           0.29,
    "contract_symbol": "TSCO260619P00195000",
    "bid":             3.10,
    "ask":             3.30,
    "mid":             3.20,
    "open_interest":   1240,
    "limit_price":     3.20,
    "signal_score":    61.4,
    "iv_rank":         100.0
  }
]
```

**Lifecycle:** Fully replaced by each selector run. Cleared by the executor after processing.
If executor is not run (auto_entry disabled), entries persist until the next selector run.

## `positions_state.json`

**Purpose:** The live trading ledger — all open and closed positions.
The most critical file in the system. Loss of this file means the monitor cannot
track what needs to be closed.

**Structure:**
```json
{
  "open": [
    {
      "symbol":          "TSCO",
      "strategy":        "CSP",
      "contract_symbol": "TSCO260619P00195000",
      "entry_date":      "2026-04-28",
      "entry_premium":   3.20,
      "entry_price":     214.50,
      "strike":          195.0,
      "expiry":          "2026-06-19",
      "dte_at_entry":    52,
      "contracts":       1,
      "iv_rank_at_entry": 100.0,
      "rsi_at_entry":    20.5,
      "alpaca_order_id": "abc123"
    }
  ],
  "closed": [
    {
      "symbol":          "TSCO",
      "strategy":        "CSP",
      "contract_symbol": "TSCO260619P00195000",
      "entry_date":      "2026-04-28",
      "exit_date":       "2026-05-12",
      "hold_days":       14,
      "entry_premium":   3.20,
      "exit_premium":    1.60,
      "pnl_pct":         0.50,
      "exit_reason":     "profit_target",
      "iv_rank_at_entry": 100.0,
      "rsi_at_entry":    20.5,
      "regime":          "bull"
    }
  ]
}
```

**`exit_reason` values:**
- `profit_target` — closed at 50% profit (or configured threshold)
- `loss_limit` — closed at 2× premium (loss limit hit)
- `dte_reached` — closed because DTE ≤ `close_at_dte`
- `rsi_recovery` — closed because underlying RSI recovered above 50
- `manual` — closed by operator outside the system

**Lifecycle:** Written by executor on entry; updated by monitor on exit.
**Backup recommendation:** Copy this file before any manual intervention.

## `options_signal_quality.json`

**Purpose:** Output of the signal analyzer — scored candidates plus closed-position
outcome statistics. The optimizer reads this file.

**Structure:**
```json
{
  "generated_at":   "2026-04-25T21:14:15+00:00",
  "regime":         "bull",
  "sell_zone_pct":  68.2,
  "data_quality":   "hv30_proxy+3d_real",
  "config_snapshot": { "...": "..." },
  "candidates": [
    {
      "symbol":       "TSCO",
      "score":        61.4,
      "iv_rank":      100.0,
      "rsi":          20.5,
      "premium_pct":  3.2,
      "annual_yield": 23.0
    }
  ],
  "iv_distribution": {
    "<40":    { "count": 164, "pct": 32.0 },
    "40-55":  { "count": 87,  "pct": 17.0 },
    "55-70":  { "count": 112, "pct": 21.9 },
    "70-85":  { "count": 89,  "pct": 17.4 },
    "85-100": { "count": 60,  "pct": 11.7 }
  },
  "outcome_stats": {
    "n":             0,
    "status":        "no_data",
    "win_rate_pct":  null,
    "avg_pnl_pct":   null,
    "avg_hold_days": null,
    "ann_yield_pct": null,
    "exit_reasons":  {},
    "by_iv_rank":    {}
  }
}
```

**`data_quality` values:**
- `real_iv` — 30+ real snapshot IV days exist (≥ 1 month post-backfill)
- `hv30_proxy+Nd_real` — N real snapshot days accumulated; HV30 proxy still dominates

**Lifecycle:** Fully replaced each signal analyzer run.

## `options_improvement_report.json`

**Purpose:** The optimizer's full output — insights, applied changes, and audit trail.
Intended for human review and the dashboard.

**Structure:**
```json
{
  "generated_at":         "2026-04-25T21:14:16+00:00",
  "pipeline_phase":        3,
  "n_closed_positions":    0,
  "min_for_insights":      10,
  "min_for_changes":       50,
  "auto_optimize":         false,
  "status":                "0 closed positions — pipeline live, insights activate at 10",
  "outcome_stats":         { "...": "..." },
  "current_insights":      [],
  "applied_this_run":      [],
  "all_applied_changes":   [],
  "config_snapshot":       { "...": "..." },
  "sell_zone_pct":         68.2,
  "regime":                "bull",
  "candidates_scored":     [ "..." ]
}
```

**`current_insights` entry structure:**
```json
{
  "param":      "iv_rank_min_sell",
  "current":    40,
  "suggested":  45,
  "direction":  "raise",
  "reason":     "40-55 IV-rank bucket win rate 30% < 40% threshold (n=8)",
  "confidence": "low"
}
```

**`all_applied_changes` entry structure:**
```json
{
  "param":  "profit_target_pct",
  "from":   0.50,
  "to":     0.60,
  "reason": "Avg hold only 8 days — raising profit target captures more theta decay.",
  "at":     "2026-06-15T16:35:00+00:00"
}
```

**Lifecycle:** Fully replaced each optimizer run. `all_applied_changes` accumulates
by merging the previous report's list with any new applied changes.
